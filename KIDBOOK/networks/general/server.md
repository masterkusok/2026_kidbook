# Что такое сервер и где он находится

Каждый раз, когда ты открываешь сайт, смотришь видео на YouTube или играешь в онлайн-игру, где-то в мире работает **сервер** — он слышит твой запрос и отвечает на него. Что это за устройство? Где оно стоит? Как оно справляется с миллионами запросов одновременно?

<!-- Картинка: стойка с серверами в дата-центре — ряды блестящих чёрных блоков с мигающими лампочками -->

---

## Сервер — это просто компьютер?

По сути — да. **Сервер** — это компьютер, который **обслуживает запросы** других компьютеров. Слово «сервер» происходит от английского *server* — «тот, кто обслуживает, официант».

Но есть важные отличия от обычного домашнего компьютера:

| Обычный компьютер | Сервер |
|-------------------|--------|
| Используется одним человеком | Обслуживает тысячи запросов одновременно |
| Включают и выключают по необходимости | Работает **24/7** без остановок |
| Есть монитор, клавиатура | Обычно **нет монитора** — управление по сети |
| Важна скорость для игр/видео | Важна **надёжность и стабильность** |
| Обычный жёсткий диск | Множество жёстких дисков с **резервированием** |
| Один блок питания | Два и более блоков питания (если один сломается — работает второй) |

> **Знаешь ли ты?** Слово «сервер» имеет два значения. **Сервер-железо** — это физическое устройство. **Сервер-программа** — это программное обеспечение, которое принимает запросы и отвечает на них. Например, Nginx — это программа-сервер, которая может работать даже на твоём ноутбуке.

## Что делает веб-сервер

Когда ты открываешь сайт, браузер отправляет [HTTP-запрос](../http_https/http_https.md) на **веб-сервер**. Задача веб-сервера:

1. **Принять** запрос от браузера
2. **Разобраться**, что именно просят (`GET /index.html`)
3. **Найти или сформировать** нужные данные
4. **Отправить ответ** обратно браузеру

Звучит просто. Но хороший веб-сервер делает это для **тысяч пользователей одновременно**, не теряя ни одного запроса.

### Статические и динамические сайты

**Статический сайт** — это набор готовых файлов: HTML, CSS, картинки. Сервер просто находит файл и отдаёт его. Как библиотека: пришёл, попросил книгу, тебе её дали.

**Динамический сайт** — сервер **формирует страницу** под конкретного пользователя в момент запроса. Страница твоего профиля ВКонтакте или результаты поиска Google — всегда разные, они генерируются на лету.

```
Статический запрос:
Браузер → "Дай мне /about.html" → Сервер ищет файл → Отдаёт файл

Динамический запрос:
Браузер → "Покажи профиль пользователя 12345"
→ Сервер запрашивает данные из базы данных
→ Подставляет данные в шаблон
→ Генерирует HTML прямо сейчас
→ Отдаёт браузеру
```

## Программы-веб-серверы

На физическом сервере работает программа, которая обрабатывает HTTP-запросы. Самые популярные:

| Программа | Кто создал | Особенности |
|-----------|-----------|-------------|
| **Nginx** | Игорь Сысоев (Россия, 2004) | Очень быстрый, эффективный, популярный |
| **Apache** | Apache Foundation (1995) | Старейший, очень гибкий |
| **Caddy** | Matt Holt (2015) | Автоматически настраивает HTTPS |
| **IIS** | Microsoft | Для серверов на Windows |

Сегодня больше половины всех сайтов в мире работают под управлением **Nginx** или **Apache**.

> **Знаешь ли ты?** Nginx придумал и написал российский программист Игорь Сысоев. В 2002 году он создавал его для сайта Rambler.ru, которому нужно было обрабатывать 500 миллионов HTTP-запросов в сутки. Сегодня Nginx используют Netflix, Airbnb, GitHub, Instagram и тысячи других сервисов.

## Виды серверов

Не все серверы — веб-серверы. Разные серверы выполняют разные задачи:

| Тип сервера | Что делает | Пример |
|-------------|-----------|--------|
| **Веб-сервер** | Отдаёт HTML-страницы браузеру | Nginx, Apache |
| **Сервер базы данных** | Хранит и выдаёт данные | MySQL, PostgreSQL |
| **Файловый сервер** | Хранит и раздаёт файлы | FTP-сервер |
| **Почтовый сервер** | Принимает и отправляет email | Gmail-серверы Google |
| **DNS-сервер** | Переводит имена в IP-адреса | [DNS](../dns/README.md) <!-- TODO: статья пишется --> |
| **Игровой сервер** | Синхронизирует состояние игры | Серверы Minecraft |
| **Стриминговый сервер** | Раздаёт видео по частям | YouTube, Netflix |

Когда ты смотришь видео на YouTube, браузер общается сразу с несколькими серверами: один отдаёт HTML-страницу, другой — само видео, третий — комментарии, четвёртый — рекламу.

## Где физически находятся серверы

### Дата-центр: дом для серверов

Серверы стоят в специальных зданиях — **дата-центрах** (Data Center, ЦОД — Центр Обработки Данных).

<!-- Картинка: вид сверху или снаружи крупного дата-центра — большое промышленное здание, много кондиционеров на крыше -->

Дата-центр — это не просто склад с компьютерами. Это сложная инженерная система:

**Питание:**
- Несколько независимых линий электропитания от разных подстанций
- Мощные генераторы на дизельном топливе — на случай отключения света
- **UPS** (источники бесперебойного питания) — дают время запустить генераторы

**Охлаждение:**
- Серверы выделяют огромное количество тепла
- Мощные системы кондиционирования работают круглосуточно
- Некоторые дата-центры строят рядом с реками или озёрами для охлаждения водой
- **Google** экспериментирует с дата-центрами прямо на морских платформах

**Безопасность:**
- Пропускной режим, охрана, видеонаблюдение
- Биометрические считыватели на входе
- Серверы физически не доступны посторонним
- Дублирование всего: два интернет-провайдера, два блока питания, два кондиционера

**Связь:**
- Несколько независимых оптоволоконных кабелей
- Подключение к **IX** (Internet Exchange) — точкам обмена трафиком, где пересекаются сети разных провайдеров

### Серверные стойки

Внутри дата-центра серверы не стоят просто на полках. Их устанавливают в специальные **серверные стойки** (racks) высотой 2 метра. Размер серверов измеряется в **юнитах** (U): 1U = 4.45 см. В одну стойку помещается до 42U — то есть 42 сервера высотой 1U.

```
┌──────────────┐
│  Сервер 1U   │  ← Например, веб-сервер
├──────────────┤
│  Сервер 1U   │  ← База данных
├──────────────┤
│  Сервер 2U   │  ← Сервер хранения данных
├──────────────┤
│  Коммутатор  │  ← Сетевое оборудование
│     1U       │
└──────────────┘
```

## CDN — серверы везде

Представь: пользователь из Владивостока открывает сайт, серверы которого стоят в Москве. Данные должны пройти 9 000 км. Скорость света в оптоволокне — около 200 000 км/с. Только физически этот путь занимает ~45 мс в одну сторону и 90 мс туда-обратно. А ещё есть маршрутизаторы, задержки... В итоге сайт будет открываться медленно.

Решение — **CDN** (Content Delivery Network, сеть доставки контента). CDN — это сотни серверов, расставленных по всему миру. Когда ты запрашиваешь файл, CDN отдаёт его с **ближайшего** к тебе сервера.

<!-- Картинка: карта мира с точками CDN-серверов, расставленными по разным городам, и стрелками к пользователям -->

| Без CDN | С CDN |
|---------|-------|
| Один сервер, один регион | Сотни серверов по всему миру |
| Долго для удалённых пользователей | Быстро для всех |
| Легко «завалить» трафиком | Нагрузка распределена |
| Один точка отказа | Отказ одного узла незаметен |

Крупнейшие CDN-провайдеры: **Cloudflare**, **Akamai**, **Fastly**. Они обрабатывают огромную долю всего интернет-трафика в мире.

> **Знаешь ли ты?** Когда YouTube начинал в 2005 году, серверы стояли в гараже Чада Хёрли — одного из основателей. За несколько месяцев трафик вырос настолько, что им пришлось срочно переезжать в настоящий дата-центр. Сейчас YouTube каждую минуту загружают более 500 часов нового видео.

## Облачные серверы

Раньше компании покупали или арендовали физические серверы. Сегодня большинство используют **облачные сервисы**: AWS (Amazon), Google Cloud, Microsoft Azure, Yandex Cloud.

В облаке ты не берёшь в аренду конкретный физический сервер — ты берёшь **вычислительную мощность**. Можно включить 1000 серверов на час и выключить — заплатишь только за этот час. Это называется **эластичность**: ресурсы масштабируются под текущую нагрузку.

Например, магазин может запустить дополнительные серверы в Чёрную пятницу, когда покупателей в 10 раз больше обычного, а потом вернуть их.

## Отказоустойчивость — как сервер не ломается

Крупные сервисы (Google, Facebook, YouTube) не могут позволить себе «упасть» даже на минуту — это миллионы рублей убытков и миллионы недовольных пользователей.

Для защиты от сбоев используют:

**Репликация** — все данные хранятся в нескольких копиях на разных серверах. Если один сломался — данные не теряются.

**Load Balancing (балансировка нагрузки)** — специальная программа распределяет запросы между несколькими серверами. Ни один не перегружается.

**Резервные дата-центры** — копия всей инфраструктуры в другом городе или стране. Если дата-центр в Москве отключат — заработает резерв в Петербурге.

**Мониторинг** — системы следят за каждым сервером 24/7 и мгновенно сообщают о проблемах.

> **Знаешь ли ты?** У Amazon Web Services были громкие аварии, когда из-за одной ошибки инженера падали сотни тысяч сайтов и приложений по всему миру. Даже самые надёжные системы иногда ломаются — именно поэтому резервирование так важно.

## Интересные факты

- **Google** потребляет около **15 миллиардов** киловатт-часов электроэнергии в год — столько же, сколько небольшая страна.
- **Facebook** (Meta) хранит более **100 петабайт** фотографий. 1 петабайт = 1 000 000 гигабайт.
- **Подводные кабели** длиной десятки тысяч километров соединяют дата-центры на разных континентах — по ним идёт весь межконтинентальный интернет-трафик.
- **Microsoft** тестировала подводный дата-центр — контейнер с серверами на дне Северного моря у берегов Шотландии. Морская вода идеально охлаждает серверы.
- **Первый сервер** (компьютер NeXT) до сих пор хранится в ЦЕРНе. На нём написано от руки: «This machine is a server. DO NOT POWER IT DOWN!!» — «Это сервер. НЕ ВЫКЛЮЧАТЬ!!»
